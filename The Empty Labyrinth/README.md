# Readme: Практика «Пустой Лабиринт»

## Цель задания

Разработать алгоритм для прохождения заранее известных типов лабиринтов вслепую. Основная сложность заключается не в навигации по неизвестной местности, а в **четком планировании и выполнении заранее составленного маршрута**, а также в написании **чистого, самодокументируемого и легко читаемого кода** в рамках строгих ограничений.

## Описание задачи

Перед вами — серия лабиринтов. Каждый лабиринт представляет собой пустую комнату, окруженную стенами. Вам известны только его тип (например, "Empty") и размеры (`width` и `height`).

**Ваша задача:** запрограммировать робота так, чтобы он дошел от стартовой позиции **`(1, 1)`** до финишной **`(width-2, height-2)`**.

**Особенности:**
- Робот не имеет сенсоров. Он не может "видеть" стены или исследовать лабиринт
- Весь путь должен быть построен исключительно на основе знания типа лабиринта и его размеров
- Лабиринт "Empty" не содержит внутренних стен, что делает задачу тривиальной с алгоритмической точки зрения. Фокус — на качестве кода

## Ключевые ограничения и философия чистого кода

Ограничения введены не чтобы усложнить жизнь, а чтобы привить лучшие практики разработки:

1. **Максимум один цикл в методе** — заставляет дробить логику на маленькие, понятные шаги
2. **Метод не длиннее 6 строк** — делает код обозримым и заставляет задуматься о его декомпозиции
3. **Запрет на `catch`** — encourages handling errors through design, not just try-catch
4. **Понятные имена методов и аргументов** — это основа самодокументируемости

**Как должно быть:**
Каждый метод должен быть атомарной операцией с именем, которое точно описывает его действие. Читатель должен понимать, *что* делает метод, глядя только на его сигнатуру, без необходимости изучать реализацию.

## Стратегия решения для лабиринта "Empty"

Поскольку лабиринт пустой, путь от старта до финиша — это простой зигзаг.

**Алгоритм:**
1. Пройти до правой стены
2. Развернуться и пройти до нижней стены
3. Повторять шаги 1-2, пока робот не окажется в целевой клетке

Однако, реализовать этот алгоритм нужно через набор маленьких, понятных методов.
